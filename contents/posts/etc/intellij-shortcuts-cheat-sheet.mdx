IntelliJ IDEA를 쓰면서 마우스 없이 키보드만으로 작업하는 순간이 늘어날수록 생산성이 확실히 올라간다.

키보드 UI를 통해 어떤 키 조합인지 시각적으로 확인할 수 있도록 구성했다.

## 키 표기 규칙

시작하기 전에, Mac과 Windows의 키 대응 관계를 알아두면 단축키를 외우기 훨씬 쉽다.

| Mac | Windows | 역할 |
|-----|---------|------|
| <Kbd>⌘</Kbd> Command | <Kbd>Ctrl</Kbd> | 주요 수정자 키 |
| <Kbd>⌥</Kbd> Option | <Kbd>Alt</Kbd> | 보조 수정자 키 |
| <Kbd>⌃</Kbd> Control | (없음) | Mac 고유 수정자 |
| <Kbd>⇧</Kbd> Shift | <Kbd>Shift</Kbd> | 확장/반전 |

:::tip
Mac에서 ⌘ 에 해당하는 키가 Windows에서는 대부분 Ctrl 이다.
이 패턴만 기억해도 절반 이상의 단축키를 유추할 수 있다.
:::

## 1. 검색 (Search)

가장 먼저 손에 익혀야 할 단축키다. 코드베이스가 커질수록 검색 없이는 작업이 불가능하다.

| 기능 | Mac | Windows / Linux |
|------|-----|-----------------|
| 전체 검색 (Search Everywhere) | <KeyCombo keys="Shift Shift" /> | <KeyCombo keys="Shift Shift" /> |
| 클래스 검색 | <KeyCombo keys="⌘ O" /> | <KeyCombo keys="Ctrl N" /> |
| 파일 검색 | <KeyCombo keys="⌘ ⇧ O" /> | <KeyCombo keys="Ctrl ⇧ N" /> |
| 심볼 검색 | <KeyCombo keys="⌘ ⌥ O" /> | <KeyCombo keys="Ctrl ⇧ Alt N" /> |
| Action 검색 | <KeyCombo keys="⌘ ⇧ A" /> | <KeyCombo keys="Ctrl ⇧ A" /> |
| 파일 내 텍스트 검색 | <KeyCombo keys="⌘ F" /> | <KeyCombo keys="Ctrl F" /> |
| 파일 내 텍스트 변경 | <KeyCombo keys="⌘ R" /> | <KeyCombo keys="Ctrl R" /> |
| 프로젝트 전체 텍스트 검색 | <KeyCombo keys="⌘ ⇧ F" /> | <KeyCombo keys="Ctrl ⇧ F" /> |
| 프로젝트 전체 텍스트 변경 | <KeyCombo keys="⌘ ⇧ R" /> | <KeyCombo keys="Ctrl ⇧ R" /> |

:::note
Shift 를 두 번 빠르게 누르는 Search Everywhere 는 IntelliJ에서 가장 강력한 검색이다.
클래스, 파일, 심볼, 액션 어디든 빠르게 이동할 수 있다.
:::

## 4. 멀티 커서 & 선택 (Multi Cursor & Selection)

반복되는 수정 작업을 한 번에 처리할 수 있다.

| 기능 | Mac | Windows / Linux |
|------|-----|-----------------|
| 멀티 커서 (위/아래) | <Kbd>⌥</Kbd> 두 번 + <Kbd>↑</Kbd><Kbd>↓</Kbd> | <Kbd>Ctrl</Kbd> 두 번 + <Kbd>↑</Kbd><Kbd>↓</Kbd> |
| 다음 동일 단어에 커서 추가 | <KeyCombo keys="⌃ G" /> | <KeyCombo keys="Alt J" /> |
| 모든 동일 단어에 커서 추가 | <KeyCombo keys="⌃ ⌘ G" /> | <KeyCombo keys="Ctrl ⇧ Alt J" /> |
| 열 선택 모드 토글 | <KeyCombo keys="⌘ ⇧ 8" /> | <KeyCombo keys="Alt ⇧ Insert" /> |

:::tip
멀티 커서는 ⌥(Option)을 두 번 누른 후 손을 떼지 않고 ↑↓로 커서를 추가하는 방식이다.
변수명을 일괄 변경할 때 ⌃+G (Mac) 또는 Alt+J (Win)를 반복해서 누르면 하나씩 선택된다.
:::

## 6. 리팩터링 (Refactoring)

IntelliJ의 가장 강력한 기능 중 하나다. 안전한 코드 변경을 보장한다.

| 기능 | Mac | Windows / Linux |
|------|-----|-----------------|
| 리팩터링 메뉴 | <KeyCombo keys="⌃ T" /> | <KeyCombo keys="Ctrl Alt ⇧ T" /> |
| 이름 변경 (Rename) | <KeyCombo keys="⇧ F6" /> | <KeyCombo keys="⇧ F6" /> |
| 변수 추출 (Extract Variable) | <KeyCombo keys="⌘ ⌥ V" /> | <KeyCombo keys="Ctrl Alt V" /> |
| 상수 추출 (Extract Constant) | <KeyCombo keys="⌘ ⌥ C" /> | <KeyCombo keys="Ctrl Alt C" /> |
| 필드 추출 (Extract Field) | <KeyCombo keys="⌘ ⌥ F" /> | <KeyCombo keys="Ctrl Alt F" /> |
| 메소드 추출 (Extract Method) | <KeyCombo keys="⌘ ⌥ M" /> | <KeyCombo keys="Ctrl Alt M" /> |
| 파라미터 추출 (Extract Parameter) | <KeyCombo keys="⌘ ⌥ P" /> | <KeyCombo keys="Ctrl Alt P" /> |
| 인라인 (Inline) | <KeyCombo keys="⌘ ⌥ N" /> | <KeyCombo keys="Ctrl Alt N" /> |
| 시그니처 변경 | <KeyCombo keys="⌘ F6" /> | <KeyCombo keys="Ctrl F6" /> |

:::note
리팩터링 단축키는 패턴이 있다. Mac에서 ⌘ + ⌥ + 알파벳 조합이다.
V=Variable, C=Constant, M=Method, P=Parameter, F=Field, N=iNline 처럼 앞 글자만 기억하면 된다.
:::

## 키보드에서 직접 확인하기

단축키는 글로만 보기보다 키 위치를 눈으로 익히는 편이 훨씬 빠르게 손에 붙는다.
아래에서 카테고리와 단축키를 선택하면 키보드 위에 해당 키가 하이라이트된다.
macOS / Windows 버튼으로 OS도 전환할 수 있으니, 자신이 쓰는 환경에 맞춰 확인해보자.

<KeyboardVisual />

---

:::success
<b>정리</b>
✔ IntelliJ 단축키는 한꺼번에 외우기보다 패턴을 먼저 이해하는 편이 훨씬 수월하다. <br/>
✔ 리팩터링은 ⌘+⌥+알파벳, 검색은 ⌘+⇧+알파벳 형태로 규칙이 있어 앞 글자만 기억하면 된다. <br/>
✔ Mac의 ⌘ 은 Windows의 Ctrl 에 대응되는 경우가 많아, 이 한 가지 패턴만 알아도 절반 이상은 유추할 수 있다. <br/>
✔ 단축키를 빠르게 익히고 싶다면 Key Promoter X 플러그인을 함께 사용하는 편이 좋다. <br/>
✔ 전체 단축키를 한 번에 보고 싶을 때는 Help > Keymap Reference 를 열어 공식 PDF로 확인할 수 있다.
:::
